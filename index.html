Hackathon Guidelinesüìú

Team Formation:

Each team should consist of 3-4 members. The team should have a common interest or project in mind.

Team Size:

Participants can form teams of 3 or 4 members.

Class Restriction:

Teams should be formed within the class.

Role of Each Team Member:

The role of each team member should be defined clearly.

GitHub Activity:

Each team member should have a GitHub account. It is mandatory.

Repository Creation:

Each team must create a GitHub repository for their project.

Commit Frequency:

Teams should commit their code regularly to document their progress and collaboration.

Documentation:

The repository should include:

  • A README.md file with an overview of the project, setup instructions, and usage information.
  • Separate folders for source code, assets, and documentation.
  • A commit history showing contributions from all team members.
